# Project DNA – Project OverviewProject DNA is an AI-powered architectural governance platform designed to preserve the technical and business knowledge of software projects throughout their entire lifecycle. Its primary objective is to transform project knowledge into structured, persistent, and machine-readable intelligence that enables AI coding assistants to understand the project as if they were long-term members of the engineering team.The platform addresses one of the biggest challenges introduced by modern AI-assisted software development: the loss of architectural context. Large Language Models are capable of generating high-quality code, but they typically lack an understanding of the project's architecture, business objectives, engineering standards, historical decisions, and domain-specific knowledge. As projects evolve, this often results in architectural inconsistencies, duplicated patterns, technical debt, and AI hallucinations.Project DNA solves this problem by creating a persistent knowledge layer that captures both the technical reality and the business intent of a software project. Rather than treating every AI interaction as an isolated conversation, Project DNA builds a continuously evolving architectural memory that every future AI interaction can leverage.The platform operates as a command-line application (CLI) intended to be executed directly inside software projects. During initialization, the command `pdna init` performs an intelligent scan of the repository. It analyzes the project's structure, configuration files, package managers, dependencies, source code organization, frameworks, programming languages, runtimes, build tools, and other technical signals. Using these observations, Project DNA identifies the project's technology stack and generates an initial knowledge base stored locally inside the `.pdna` directory.This knowledge base contains structured documents describing different aspects of the project, including architecture, detected technologies, dependencies, coding conventions, security rules, API conventions, business context, domain context, architectural decisions, scanner reports, and other metadata required to support future AI reasoning. These documents become the project's persistent architectural memory.After initialization, developers can execute the `pdna project overview` command to provide a comprehensive human-written description of the project. This overview represents the business vision of the product and explains what the system is intended to solve, who its target users are, its functional capabilities, business objectives, domain concepts, constraints, and any additional context that cannot be inferred solely from source code.The project overview is considered the primary source of truth for business knowledge. While repository scanning accurately represents the technical implementation, only the human overview can fully explain the product's purpose, business rules, expected behaviors, stakeholder expectations, and domain language. Combining these two perspectives allows Project DNA to build a significantly richer understanding of the project.Once the overview is provided, Project DNA sends the collected information to an AI reasoning engine powered by Fireworks AI. The AI does not generate source code. Instead, its responsibility is to analyze, reason, infer, validate, and organize the project's architectural knowledge. It extracts business intent, identifies business domains, recognizes technical domains, detects architectural patterns, evaluates risks, identifies missing information, and generates structured architectural insights that can be consumed by future AI workflows.The structured output generated by the AI is validated against predefined schemas before being persisted locally. This guarantees consistency, predictability, and compatibility across future versions of the platform.Project DNA distinguishes different categories of knowledge.Business Context represents the product's business vision. It describes why the project exists, the problems it solves, the intended users, the business goals, the overall product value, and the major business capabilities delivered by the system.Domain Context represents the project's logical domains. Each domain corresponds to a distinct business area or responsibility inferred from the product vision. Domains provide a structured representation of the application's functional organization and establish a shared vocabulary for both humans and AI systems.Architecture Insights represent the AI's complete architectural reasoning process. They summarize architectural styles, important modules, technology choices, dependency intentions, implementation constraints, architectural risks, missing context, recommendations, confidence scores, and supporting evidence extracted from both the repository and the project overview.One of Project DNA's core principles is provider abstraction. The AI layer is completely decoupled from any specific Large Language Model provider. Although Fireworks AI is currently used as the reasoning engine, the architecture is designed around provider interfaces, allowing additional providers such as OpenAI, Anthropic, Gemini, or local models to be integrated without affecting the core application.Another important principle is local-first architecture. All generated knowledge is stored inside the project itself rather than in external databases. This approach improves transparency, portability, version control integration, developer trust, and long-term maintainability while keeping operational costs low.The platform is intentionally designed as a foundational intelligence layer rather than a code generation tool. Its responsibility is to provide reliable architectural knowledge that other AI agents can consume. Future capabilities will include intelligent architectural validation, context-aware prompt generation, architectural governance, semantic knowledge graphs, project versioning, multi-provider orchestration, contextual retrieval, and advanced reasoning workflows.Ultimately, Project DNA aims to become the architectural memory of every software project. By preserving business knowledge, technical context, architectural decisions, and engineering standards in a structured format, it enables AI assistants to generate more accurate, context-aware, and architecturally consistent solutions while significantly reducing hallucinations and preserving long-term software quality.
